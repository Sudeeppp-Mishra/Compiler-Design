# Lab 4 – Lexical Analyzer using Flex

## Objective

The objective of this lab is to implement a lexical analyzer using **Flex (Lex)** that recognizes integer numbers, arithmetic operators, parentheses, whitespace, and newline characters. It also reports any invalid characters encountered during lexical analysis.

---

## Files

| File                               | Description                                        |
| ---------------------------------- | -------------------------------------------------- |
| `key.l`                          | Flex source file containing lexical analyzer rules |
| `lex.yy.c`                       | C source file automatically generated by Flex      |
| `a.out` / `lexer` / `result` | Executable generated after compilation             |

---

## Tokens Recognized

The lexer recognizes the following tokens:

- Integer numbers (`0-9`)
- Addition (`+`)
- Subtraction (`-`)
- Multiplication (`*`)
- Division (`/`)
- Left Parenthesis (`(`)
- Right Parenthesis (`)`)
- Newline (`\n`)
- Whitespace (ignored)

Invalid characters are reported as lexical errors.

---

## Program Logic

The lexer performs the following operations:

1. Reads input from the user.
2. Recognizes integer values.
3. Returns arithmetic operators and parentheses as individual tokens.
4. Ignores spaces and tabs.
5. Detects newline characters.
6. Prints an error message for unknown characters.

---

## Compilation

Generate the lexer:

```bash
flex key.l
```

Compile the generated C file:

```bash
cc lex.yy.c -o result
```

Run the program:

```bash
./result
```

---

## Sample Input

```
12 + 45 * (8 - 3)
```

### Sample Output

The lexer returns the corresponding sequence of tokens for:

- Numbers
- Operators
- Parentheses

If an invalid character is entered, for example:

```
12 + @
```

Output:

```
Unknown character: @
```

---

## Regular Expressions Used

| Regular Expression | Meaning                              |
| ------------------ | ------------------------------------ |
| `[0-9]+`         | Integer numbers                      |
| `[ \t]+`         | Ignore spaces and tabs               |
| `\n`             | Newline character                    |
| `[()+\-*/]`      | Arithmetic operators and parentheses |
| `.`              | Any invalid character                |
